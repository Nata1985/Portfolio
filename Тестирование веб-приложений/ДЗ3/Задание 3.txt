Тестирование веб-приложений
Урок 3. Особенности тестирования веб-приложений
1. Найти баги в форме.
Найдите баги в форме. На форме есть как минимум десять багов, которые надо найти и описать. Формат сдачи: текстовый файл (pdf, Google Docs) или таблица, например, Google Sheets. А формат описания багов вам известен с предыдущего семестра. Возможно, вы найдёте больше десяти багов — это хорошо.
2. Построить полный путь по относительному.
Этот сайт даётся как пример, его в реальности не существует! Пользователь находится на странице, которая посвящена одной из пород кошек и имеет такой адрес: https://animals.verybigencyclopedia.com/catalog/mammals/felidae/cats/british-shorthair/index.html. На странице есть относительные ссылки, где в параметрах href указано:
//verybigencyclopedia.com/about/.
/images/cats/british-cat.jpg.
/scripts/style-cats.css.
../ragdoll.
../images/domestic-cat.jpg.
../../../felinae/.
images/cats/photo-british-cat-01.jpg.
verybigencyclopedia.com/catalog/mammals.
#references.
../british-shorthair/gallery.
Для каждой ссылки напишите абсолютный путь.
Формат сдачи: текстовый файл (pdf, Google Docs).
3. Проверить отправку формы при отключённой валидации на клиенте.
Воспользуйтесь инструкцией из раздела «Обход клиентской валидации» и сделайте то же самое, но для страницы «Вход» сайта GeekBrains.
Формат сдачи: скриншоты, на которых демонстрируется изменённый код в DevTools, результат отправки формы с отключённой клиентской валидацией; скриншот из Fiddler/Charles, он покажет, что запрос действительно ушёл без этих полей. Красным выделено то, на что надо обратить внимание:
выбран верный запрос слева;
раздел WebForms — во вкладке Inspectors;
значения полей пустые;
ответ сервера — 200 OK.
4. Зайти на сайт через подстановку cookie.
Используйте инструкцию из раздела «Подмена cookie» и зайдите на http://testingcourse.ru/docs или на https://crm.geekbrains.space в обход страницы логина. Данные для логина:
user / Password! — testingcourse/docs
user / 1234 — Aplana CRM
Формат сдачи: четыре скриншота — по два из каждого браузера. Один скриншот показывает сайт с залогиненным пользователем, второй — DevTools с cookie. Важно, чтобы значения cookie было видно, в обоих браузерах они должны быть одинаковые.
